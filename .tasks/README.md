# Bookstore 开发任务目录

`.tasks` 是 bookstore 项目的产品需求文档目录，用于描述业务功能变更、技术改进等需求。

## 目录结构

```text
.tasks/
├── README.md
├── backlog/          # 待实施的需求
├── active/           # 正在推进的需求
└── done/             # 已完成需求的归档记录
```

## 文档状态变化

任务状态变化时移动文档所在目录：

- 待开始的任务存放在 `backlog/`
- 开始实施后移动到 `active/`
- 完成并记录验证结果后移动到 `done/`

## 文件命名

任务文件使用三位序号加简短英文 slug：

```text
001-frontend-test-coverage.md
002-oauth-client-hardening.md
```

## 使用约定

- 开始实现具体任务前，先读取对应任务文档。
- 实施中发现范围变化时，更新任务文档的目标、非目标、实施步骤或状态记录。
- 这里的内容是协作和规划资料，不属于业务运行时代码、测试夹具或构建输入。
